# Windows Logging for SOC

## Overview
This write-up summarizes what I learned while completing the **Windows Logging for SOC** room on TryHackMe.
The room introduces Windows Event Logs and explains how they are used by Security Operations Center (SOC) analysts to monitor systems, investigate incidents, and identify suspicious activity.

## Objectives
- Understand the purpose of Windows Event Logs.
- Learn the different log categories.
- Discover how SOC analysts use logs during investigations.
- Recognize the importance of log analysis in threat detection.

## Main Log Categories
- **Application** – Stores events generated by applications.
- **Security** – Stores authentication events, account changes, and other security-related activities.
- **System** – Stores events generated by Windows and device drivers.

## Skills Gained
- Understanding Windows Event Logs
- Basic log analysis
- Security monitoring concepts
- Threat detection fundamentals

## Why This Matters
Windows Event Logs are one of the most valuable sources of information during a security investigation. They help analysts detect suspicious behavior, investigate security incidents, and understand what happened on a system.

## What I Learned
After completing this room, I understand:
- Why Windows Event Logs are essential for SOC analysts.
- The purpose of the Application, Security, and System logs.
- How log analysis supports threat detection and incident response.
- The importance of monitoring Windows systems for suspicious activity.

- Windows Event Viewer

## Final Thoughts

This room provided a solid introduction to Windows logging and helped me understand how event logs are used in real-world Security Operations Centers.
